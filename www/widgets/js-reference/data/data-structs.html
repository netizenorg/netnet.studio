Data Structures

It's possible to store more than one value in the same variable. These variables become "data structures" the most fundamental of which is called an object one of the 6 "data types" discussed in the introductory notes. There's are a couple of ways to create a new object in JavaScript, the most common is to assign an empty set of curly braces {} to a variable.




The typeof operator can be used to find out what data type is stored in a variable. If we create a variable, for example let i = 0 and then check typeof i it would be "number".


Another way is to create a new "instance" of an object using the constructor method like new Object, which is less common when creating basic objects like these, instead that's used to create instances of more complex types of objects, the the kinds we get access to when using different creative coding libraries.




Typically we create objects with some data in them to start. Objects are essentially a series of key/value pairs. These internal variables are refered to as properties. An object's property has a name or key followed by a colon : and then the value of that property. We separate each property (the key/value pairs) with commas ,




There are a couple of different ways to reference a specific property within an object, the first is using the dot . syntax, the other is by passing the name of the property as a string into a pair of square brackets [].




The dot syntax is much more common, but the square brackets have their advantage. For example, if you have a properties name stored in a changing variable elsewhere in your code, you could place that variable between the square brackets to access that particular value.




We'll stick to the dot syntax for the rest of these examples as they are more common. You can reassign any of these properties like we would any other variable. We can remove a particular property from an object using the delete keyword.




When an object gets very long, it's best to write the values into multiple lines. An object's properties can contain any kind of data, including other objects (like the "city" property in the example below), We can access nested properties with the dot syntax as shown in this example.




You could even store a function inside an object. When there's a function inside of an object we typically refer to it as a method, this is what we call a function that belongs to an object rather than being it's own global variable.




Arrays

Another common data structure is called an Array. You can create one of these by accessing a set of square brackets [] to a variable. Technically, an Array is just another kind of "object" (we'll come to realize that nearly everything in JavaScript behaves like an object, but more on that later).




Like basic objects, an array can be created a couple of different ways, either by assigning a set of empty square brackets [] to a variable, or using the Array constructor new Array().




If we check to see what type of data is stored in these variables it will return "object", this is because anything we can create a new instance of is essentially some kind of object. But these special kind of objects (of which Arrays are only one) have their own built in properties and methods that we can use just by creating them. Another operator, similar to typeof which will tell us what kind of object a particular variable might be is the instanceof keyword.




Like objects, we can create arrays with data in them to start. But unlike objects, the data aren't stored in key/value pairs, instead array's are simply a list of comma separated values. To access a piece of data we must use the bracket [] syntax, passing in the index number (or position in the list) of the item we want, starting from 0. We can reassign a particular item in the array using this way as well. It's also worth noting, that although in this example the values in the array are all strings, you can technically store any combination of values (of any data type) in arrays, for example const c = [1, 'hi', true]




Arrays have their own bult-in methods (which again, is what we call functions built-in to an object). There are loads which you can learn about in the MDN documentation page, in these examples we'll just cover the basic ones. To add a new item to the end of an array's list we use the push() method, to remove an item from the end of the list we use the pop() method. The join() method turns the entire array into a string we can send to our alert pop up.




We can also add and remove items from the start of the list. To add a new item to the start of an array's list we use the unshift() method, to remove an item from the start of the list we use the shift() method.




We can use a for loop to iterate over an array. Array's have a property called length which returns the number of items currently in the array. We can use this to define the loops condition as i < names.length, this way we know our loop will only ever loop as many times as there are items in the array. We can then use the i variable as our index value for the items we want to log from the array.
