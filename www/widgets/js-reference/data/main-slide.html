Unlike HTML and CSS, JavaScript is a programming language. Programming languages come in many forms (Python, Java, C++, Rust, etc.) but at their core they all serve the same purpose: letting us give instructions to a computer. Each language shares the same basic building blocks—things like variables, functions, loops, and conditionals—but each one has its own way of writing them. That way of writing is called its syntax.


Although JavaScript is not the only programming language we can use on the Internet, it's definitely the most popular, which means there's a large community creating libraries, tutorials and answering questions online. Like any other programming language, JavaScript can be used to create all sorts of things outside of the web browser, from native apps to robots, you can even use it to script other programs like the Adobe Creative Suite and Microsoft Office. All that said, JavaScript was originally invented for the Web, in 1995 for a browser called Netscape Navigator...


"[Netscape] displayed Web pages that were not very lively. You could have a nice cartoon of a monkey on the Web page, but there was no way to make the monkey dance when you moved over it with your mouse. Figuring out how to make that happen was the job of a language developer named Brendan Eich. He sat down and in a few weeks created a language called JavaScript [...] as browsers proliferated and the Web grew from a document-delivery platform into a software-delivery platform, JavaScript became, arguably, the most widely deployed language runtime in the world. If you wrote some JavaScript code, you could run it wherever the Web was—everywhere." —Paul Ford


As Paul Ford notes, JavaScript helped turn the Web into a place where software runs. Software is just a set of instructions—like a recipe. Before the steps, a recipe lists its ingredients. In programming, our “ingredients” are data, and we store that data in variables...




Variables

A variable is a named slot in memory where you store data. Variables can hold different kinds of data (see below). In our example, the variable is named user, its type is a string, and its value is 'Ada Lovelace'.


Different Types of Data

(see Data Types docs for more info)


  • string: can be any text and must always be surrounded by quotes, these can be double quotes "hi" or single quotes 'hi', a string can be as short as 0 characters long "" (also known as an empty string) and as long as your memory allows "a rose is a rose is a rose is a rose is a rose" (much longer than that)

  • number: this can be an integer like 1337 or a float (aka decimal) like 3.141592653589793, the largest number is Infinity and the smallest number is -Infinity (sort of, big numbers can get a little complicated, see the "Data Types" docs for more on that).

  • boolean: named after George Boole, this can only be one of two values, either true or false. Deep down, inside the computers circuitry everything can ultimately be breaken down into booleans, true or false, on or off, 1 or 0, binary code.

  • objects: it's possible to store more than one value in the same variable. These variables become "Data Structures" the most fundamental of which is called an object. In some sense, almost everything in JavaScript can behave like an object. (but we'll come back to this later, refer to the "Data Structure" docs for more on that).

  • functions: instructions are also a form of data, because we need to store them in memory before we "run" or execute our instructions by passing them along to our computers CPU (central processing unit). Sometimes we'll write a small set of instructions we want to "call" or "run" at different points in our program, we call this sub-set of instructions a function. Though we store functions in there own variables, they're really not like the other types and deserve special consideration. (We'll come back to functions later).

There are a few other data types, but those are the most important to understand for now. If you'd like a deeper dive into this topic refer to the JavaScript "Data Types" docs.


When you make a new variable, you start with a declaration keyword. Back in the day there was only var, but this proved problematic for reasons we won't get into here. So in 2015 ECMA, the European Computer Manufacturers Association (the nonprofit organization that standardizes JavaScript) introduced two better options: const and let. When a variable's value is meant to stay constant and never change, we use const. We use let when we plan to reassign it. For example, declaring let year = 2015 means it can change later, year = 2016. Notice we only write the keyword when we first declare the variable, after that, we just re-assign it. A good rule of thumb: prefer const by default (it helps prevent accidental re-assignments), use let if you actually need to change the value.




A single instruction in your program is called a statement. Declaring a variable is one example of a short statement. Statements often contain an expression, a piece of code that produces a value. For example, 2 + 5 is an expression that produces the value 7. If we write let sum = 2 + 5, that whole line is a statement: we declare a variable named "sum", evaluate the expression, and store the result. Because sum was declared with let, we could later change its value if needed. Other mathematical or arithmetic operators include - for subtraction, / for division and * for multiplication. There are other operators, but we'll get to those later.

Programs normally run top to bottom, one statement after another. In the example below we start with let i = 100. On the next line we double it i = i * 2, and then add one i = i + 1. By the end, i holds the value 201. If we swapped the last two lines, adding 1 before doubling, we'd end up with 202 instead. The order of your statements matters.


Functions

Another kind of statement is a function call (aka function invocation). A function is a stored set of instructions, and calling it runs those instructions. For example, every browser has a built-in function named print() that tries to send the current page to your printer. Notice the parentheses, without them, you're just referring to the function, the variable name which stores the instructions. With the parentheses, you're telling the computer to run those instrutions.






You can send data into a function using their () parenthesis, this input data is called an arguments. For example the browser also has a built in function called alert. When we call this funcion the browser will display its alert box, whatever we pass into the alert function's argument becomes the message that appears in the pop up, click the diagram below to run that function.

You're not limited to the browser's built-in functions, you can write your own. The simplest way is with the function keyword, followed by the name you want to give it, a set of parentheses (), and a block of code inside { curly braces }. If your function should take input, you list that input as a variable inside the parentheses. For example, (x) means "this function expects a value we'll call x." Inside the braces you can then use x like any other variable.





What happens in a function, stays in the function, unless you explicitly want to send something back out. The way to do that is with the return keyword. Whatever value comes after return is handed back to the place where the function was called. This means we can save the result of a function into a variable, just like we save the result of an expression.





For example, our square function takes a number and multiplies it by itself. Let's assume the variable length in the diagram above was set to 5, if we pass that in the function calculates 5 * 5 and returns 25. We then store that result in a variable named area, which we can use later in our program.




Earlier we said programs run line by line, top to bottom. That's true by default, but functions let us change the default flow. Notice in the example above: the code x * x is written on line 2, but it doesn't run right away. It only runs later, when we call square(5) on line 5. Functions act like bookmarks in your code, you write them once, and they're only activated when you decide to call them.


It's worth mentioning that way we defined the square function above is just one way to create a new function, not all the JavaScript code you come acorss will define functions that way, so it's worth reviewing the Functions (alt syntax) docs for other variations and considerations.


Conditional Statements


Sometimes we only want code to run if a certain condition is true. That's where conditional statements come in. A condition is just a true/false question. If it's true, the code inside the { curly braces } runs. If it's false, that block is skipped. This gives us control over the program's flow, instead of always running everything in order.



In the example if (score > 7), the symbol > is a relational operator. It asks, "is the value on the left greater than the value on the right?" If the answer is true, the code in the block runs. If not, the block is skipped. Other relational operators you'll use often include:

  • < less than
  • <= less than or equal to
  • >= greater than or equal to
  • === equal to (strict comparison)
  • !== not equal to

Below we have three possible branches. First, the program checks if (score > 7). If that's true, it runs the first block and ignores the rest. If not, it moves to the else if (score > 3) check. If that's true, it runs the second block. If neither condition is true, the final else runs as a "catch-all" backup. This kind of branching is central to all algorithms, it's how programs make choices and respond differently depending on the data.




You can also write conditionals like if (name), which check to see if that varible is truthy of falsy. If a variable has a value in it (meaning it is not empty or "undefined"), it's usually going to be truthy, but there are some values which evaluate as falsey, including 0, null, '' (an empty string) and obviously false.





Loops

Sometimes we don't just want to check a condition once, we want to keep repeating some code as long as the condition stays true. That's exactly what a while loop does.




This starts by declaring a variable let i = 0. Each time the loop runs, it checks if i < 10. If that's true, it runs the block. Inside the block we use the i++, which is just shorthand for i = i + 1. In other words, it increases i by one each time. The loop keeps going until i reaches 10, at which point the condition is false and the loop stops. Some people also put semicolons at the end of every line (it's required in some other languages), but in JavaScript they're optional most of the time. The main place you must use them is when you're writing multiple statements side by side, like inside a for loop.






Where do we write it?

Just like CSS, JavaScript can be added to a page in 3 similar ways ways. The first is inline by attaching JavaScript directly to an element with event attributes like onclick. An event is something that happens in the browser, like a click, a key press, or the page finishing its load. By attaching code to an event, we tell the browser "when this happens, run that code."




Inline JavaScript works, but it quickly gets messy if you try to add more than a couple of lines. And remember, events aren't limited to attributes like onclick, we can also respond to them directly in our JavaScript code (we'll cover that in another lesson). A more common approach is to put JavaScript inside a script element, usually at the bottom of your HTML file.




The script element helps keep your HTML and JavaScript separate while staying in the same file—just like CSS in a style element. For bigger projects, you'll usually put JavaScript in its own file and load it with a script tag that points to it. Instead of writing code between the tags, use the src attribute to reference the path to your file (Just like you can move CSS into its own file and load it with a link tag).
