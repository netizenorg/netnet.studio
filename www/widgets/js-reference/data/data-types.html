A variable, like the one in the diagram below, is a named slot in memory where you store data. Variables can hold different kinds of data (which we call their type). Different programming langauges handle "types" differently, some are strongly typed which means we assign the type to the variable itself, like string x = "hello" or boolean a = true, other langauges, like JavaScript, are loosly typed (aka dynamically typed) which means the type belongs to the value, not the variable.

In other langauges this variable could be reassigned to a different value, but would have to remain a string, but becuase JavaScript is loosely or dynamically typed we can change it to anything we want, even a value of a different type. Number Types

"number"

A "number" is one of the most fundamental JavaScript data types, almost any type of data can be broken down into numbers. The pixels in an image are represented by numbers, each pixel is typically defined by three numbers (each a value between 0 and 255), which specifies how bright to make each channel: red, green and blue (the primary colors of light). An audio file is also just a seris of numbers, usually between -1 an 1, each value representing the position of the speaker at that point in time (vibrating a speaker back and fourth is how we produce sounds). Even strings, characters of text which have their own data type (see below) can be brokendown into numbers, one for each character (or symbol) in the text. We also use numbers in our logic, to set thresholds in conditional statment and for automation when determining how many times to loop something. All of which is to say, numbers are important.


In some programming languages numbers are broken up into two different types integers (whole numbers) like 1337 and floats (decimal numbers) like 3.14, but in JavaScript these are both just numbers. While it's not important to understand all the technical details behind how JavaScript accomplishes this and stores our numbers in memory, it is worth knowing that it uses 64-bits of binary code to store each of our numbers. This means there's a limit to how big our number can be (~9 quadrillion) and it also means the decimal math is approximate, for example 0.1 + 0.2 won't come out to exactly 0.3, it's actually 0.30000000000000004, so treat fractions as very close estimates.


JavaScript also has a very special values which also have a typeof "number", including Infinity and -Infinity, which sounds odd at first, but can actually be pretty useful. For example, you might want to keep track of a "max" value in a variable, maybe this value changes based on user input, but we need to give it some value to start and just need it to be something really big, bigger than the user would reset it to, the biggest possible number, what should we pick? let max = Infinity. Another special value is NaN, which stands for "not a number", it's what you get when you do something like 0 / 0 or Infinity - Infinity, the value of both would be NaN. You could check if something is NaN using the isNaN() function.



JS Quirk: keep in mind, even though NaN literally stands for "not a number", when a variable has that value, it is technically a typeof "number".


"bigint"

A BigInt, short for "big integer" is whole numbers of any size, written with an n suffix like 123n. Use this when you need integers beyond what can be stored in our 64-bit number types. It's also ideal when you need perfectly exact integer math (like in cryptography). BigInts have no decimals and don't mix directly with numbers, so if you ever need to write expressions with variables of different number types you can use the Number() or BigInt() constructor functions to convert one to the other.



Non-Number Types

"undefined"

It's possible to declare a variable, giving it a name, but not assign it any value (no = with data after it), for example let x. In these instances the typeof this variable would be "undefined". Our container (the space in memory) has a name, there's just nothing in it (yet). Another common situation where you'll encounter "undefined" is when you have a function that doesn't return any value, by default it will return undefined



"boolean"

We often need to distinguish between two states, "is dark-mode one or off? Is the user logged in? Yes or no?" In these instances it's most effecient to use a "boolean" value. In the mid-1800s, mathematician George Boole developed Boolean algebra, a way to express logic (previously seen as a purely philosophy concept) with mathematics using values that can be only one of two states: true or false. This idea later became central to digital computing: hardware represents these states as 1 and 0 (on/off). By combining this "binay code" we can build all higher-level computations. Deep down, everything a computer does reduces to these fundamental boolean choices.


"string"

Anytime we want to store a piece of text, be it a name or an entire story, we can store it as a string. Strings are made by placing the text we want to store within quotes. There are three different types of quote marks we can use, single quotes, double quotes and backticks.



When using single ' or double " quotes, you'll need to escape that same quote using a \ if it appears inside the string: 'It\'s fine' or "He said \"hi\"". You can also switch quote types to avoid escaping: "It's fine". A string can't be split across different lines of code, but you can use \n to insert a line break, for example "first line\nsecond line"). To combine strings (concatenation), you can use the + operator.


Backticks ` are special kinds of strings called template literals. They allow actual line breaks right inside the string, and they support interpolation, which means you can insert values with ${...}, which is often easier to read than lots of +.



"symbol"

A symbol is a special kind of value that's guaranteed to be unique. You create one with the Symbol() constructor function, similar in vibe to calling Number() or BigInt() (which we discussed earlier), except every call to Symbol() gives you a brand-new, unique symbol. Two symbols with the same description are still different values.



If this feels niche, that's okay, symbols are there for the rare moments when uniqueness really matters. Most of the time, you can happily ignore them. In 99% of scenarios, strings, numbers, arrays, and objects cover almost everything you need. But if you ever need a value you can compare for identity and be absolutely sure it's the exact same token (and not accidentally collide with any other value), a symbol is perfect.

null

Earlier we discussed undefined, which is the teypof value a variable has if you haven't assigned it anything, but say your variable does have a value and you want to remove it? When you want to make a variable intentionally empty, it's best practice to set it to null.



JS Quirk: technically speaking null is it's own primitive type, which is why we included it on the list in the diagram at the top of these docs, but for historical reasons there is no typeof "null", instead when you check the type of a variable set to null it returns "object". Important to keep in mind.


Objects

It's possible to store more than one value in the same variable. These variables become "data structures" the most fundamental of which is called an object. The simplest way to create an object is to write key: value pairs, each separated by a comma , within { curly braces }. You can then use the . dot notation to interact with specific values stored in that object. We call the variables (these keys) stored inside an object a property.



Objects are very special type in JavaScript, not just because they're in the non-primitive category, but because almost everything in JavaScript either is, or behaves like, an object. This is a topic that deserves it's own section. To dive deeper in the world of objects see the Data Structure docs.

Functions

Stricktly speaking, functions aren't a separate built-in data type in JavaScript, they're callable objects (objects that you can invoke or "run"). A function's body is a collection of statements, making the function itself a kind of value: you can store it in a variable and pass it around. Technically, JavaScript treats functions as objects with special behavior, but because they're so fundamentally different it's useful to know when a variable contians a function rather than another kind of object. For this reason, JavaScript's typeof operator has a handy special case, checking a variable that stores a function returns "function", instead of "object" (even though, technically, that's what it is).



In practice we use functions to package actions, name and reuse code, and to participate in control flow. It's helpful to think of them as their own concept, things you can call (or run) rather than "just another object." You'll also see the word method, which simply means "a function stored on an object." Functions are an incredibly important part of programming so there's a lot more to say about them.
