Our Code Reflects Our Values

(but don't take our word for it, audit it yourself)


Don't get me started on "privacy policies", these days online platforms extract as much data as algorithmically possible—almost entirely unregulated—while they keep their code entirely private, protected by "intellectual property" laws (code which would otherwise reveal the egregious extent of their exploitation)

Here's our policy: my creators are strong advocates of privacy and transparency, which is why they give you complete control of Your Session Data via this widget.

Any data about you that I save, whether it's personal data like your name or behavioral data like which widgets you've got opened and where you've placed them, is stored entirely locally in your browser on your computer and made accessible via the Session Data widget. NOTE: this means that if you visit netnet.studio on another browser I won't know who you are. This also means that if you are using a public computer you might want to delete this data before someone else uses it.

None of your data leaves your browser,
with a couple of deeply considered exceptions...

URL Shortener (short sharable sketches)

When you save a sketch you're given the option to either download it to your computer, upload it to GitHub (if you've connected to your account, more on that below) or generate a sharable URL. This sharable link encodes all the code in your sketch directly into the URL, meaning it's not stored or saved anywhere in the conventional sense, it exists intentionally and exclusively as a URL. Because they contain the compressed code of your sketch, those links can get pretty long. For your convenience you can opt-in to our URL shortener for shorter links to shareable sketches. If you choose to shorten a URL I'll send your code to my server to store in a data base of shortened URLs. There's no other way to shorten a URL (But feel free to disagree, if you know another way please submit a PR on GitHub), that said we don't sell or monetize these sketches in any way.

GitHub (data in transit)

The second exception happens if and when you connect me to your GitHub account to allow me to save any projects you make here as repositories (or "repos" for short) on your GitHub. In order to communicate with GitHub's servers I'll act as an intermediary, passing data along between your computer and GitHub when you decide to "push" updates to your account. Otherwise, while working on a project, all the code and assets never leave your browser. It took some clever engineering to pull that off, it would have been technically easier to store your code on our servers like most online code editors do, but that would compromise our values (again, you can refer to my open source code on GitHub for proof of this)

netnet stats (web analytics)

Lastly, we do collect some analytics in order to make sure things run smoothly and to gain some high-level insight on our traffic, things like how many visits we're getting, when, and roughly where in the world from. We use Plausible, an independent, open-source analytics company based in the EU that is fully GDPR-compliant, collects no personal data, uses no cookies, and shares nothing with third parties. We previously wrote our own custom analytics from scratch for the same reason, to avoid tools like Google Analytics that have a well-documented history of compromising user privacy (even on well-intentioned sites). Plausible shares our values and is much easier for us to maintain.
